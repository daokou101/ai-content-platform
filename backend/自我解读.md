4.4分页查询
ContentPageVO vo = contentService.getContentPage(page, size, keyword, templateType, userDetails.getUserId());
dto：前端传给后端，vo：后端传给前端。
Page<ContentVO> page = new Page<>(pageNum, pageSize);
page是一个分页对象，将page放在mapper传参的第一个位置，mybatisplus自动将这次任务划分为分页任务。例如：
                List<ContentVO> records = contentMapper.selectContentVOList(
                page, keyword, templateType, userId
                );
* Page 参数是 MyBatis-Plus 的分页模型：
    *   必须放在第一个参数位置，MyBatis-Plus 的 PaginationInterceptor 会自动识别它
    *   并自动在 SQL 前后拼接 COUNT 查询和 LIMIT 语句。
    *   首先，mybatisplus会在正式查询之前执行一次count操作，数出符合条件的总条数。
    *   并且page.getTotal()可以获取到这个count的结果。
    *   接下来执行具体的查询条件，由于是分页查询，所以有一个pageSize和pageNum参数，mybatisplus会在最后自动写上一个limit语句，实现分页查询。

4.5删除内容（逻辑删除）
前端调用control层，control调用contentService.deleteContent(id, userDetails.getUserId());。
直接调用mapper的deleteById，由于content表的deleted字段加上了@TableLogic注解，当检测到delete的时候，
底层会自动执行update操作，将id = id的这行数据的deleted字段更新为逻辑删除。
这个逻辑删除由程序员自己指定，例如：
@TableLogic(value = "0", delval = "2")
private Integer deleted;
默认为0=未删除，1=已删除
