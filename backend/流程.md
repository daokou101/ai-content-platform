  流程 1：登录/注册

  1.1 登录

  前端（Login.vue）
    → POST /api/auth/login  { username, password }
      → AuthController.login(LoginDTO)
        → authService.login(loginDTO)                        // AuthService.java:50
          → authenticationManager.authenticate(              // SecurityConfig.java:80 → @Bean
              new UsernamePasswordAuthenticationToken(
                username, password
              )
            )
            → CustomUserDetailsService.loadUserByUsername(username)  // 自动调用
              → userMapper.selectOne(...)                       // 查 sys_user 表
              → return new CustomUserDetails(user)              // 封装成 Spring Security 用户对象
            → BCryptPasswordEncoder.matches(password, 加密后的密码) // 自动密码比对
            → 返回 Authentication 对象（含 CustomUserDetails）
          → jwtUtils.generateToken(userId, username, role)    // 生成 JWT
          → redisTemplate.opsForValue().set(                  // Token 存 Redis
              "token:" + userId, token, 24h)
          → redisTemplate.opsForSet().add(                    // 加入在线用户 Set
              "online:users", userId)
          → return LoginVO(token, role, nickname, avatar...)  // 返回给前端

  1.2 注册

  前端（Register.vue）
    → POST /api/auth/register  { username, password, nickname, email }
      → AuthController.register(RegisterDTO)
        → authService.register(registerDTO)                  // AuthService.java:96
          → userMapper.selectCount( username='xxx' )          // 检查用户名是否已存在
          → passwordEncoder.encode(password)                  // BCrypt 加密
          → userMapper.insert(user)                           // 插入 sys_user
          → pointsService.addPoints(userId, 100, "注册赠送")  // 送 100 积分
          → return login(loginDTO)                            // 注册完自动登录

  流程 2：JWT 认证过滤器（每次请求都走）

  任何请求（除白名单外）
    → JwtAuthenticationFilter.doFilterInternal(request, response, filterChain)
                                                            // JwtAuthenticationFilter.java:43
      → extractToken(request)                                // 从 Authorization: Bearer xxx 提取
      → 如果没取到: request.getParameter("token")            // 从 ?token=xxx 提取（SSE 专用）
      → jwtUtils.validateToken(token)                        // 解析 + 验证 JWT（过期、签名）
      → jwtUtils.getUsernameFromToken(token)                 // 从 JWT 体里拿用户名
      → userDetailsService.loadUserByUsername(username)      // 查数据库拿完整用户信息
      → new UsernamePasswordAuthenticationToken(             // 构建 Spring Security 认证令牌
          userDetails, null, userDetails.getAuthorities())
      → SecurityContextHolder.getContext()                    // 存到线程上下文中
          .setAuthentication(authentication)
      → filterChain.doFilter(request, response)              // 继续后续过滤器

  后续任何地方通过以下方式获取当前用户：
  - @AuthenticationPrincipal CustomUserDetails userDetails（Controller 参数）
  - SecurityContextHolder.getContext().getAuthentication()（Service 层）

  流程 3：AI 内容生成（SSE 流式 — 核心亮点）

  前端（Generate.vue 点击"开始生成"）
    → new EventSource('/api/ai/generate/sse?keywords=xxx&templateType=article&token=xxx')
            ↑
            EventSource 无法设置请求头，所以 JWT 通过 URL 查询参数传递
            JwtAuthenticationFilter 中通过 request.getParameter("token") 获取

       ====== 第一步：Controller 建立 SSE 连接（主线程） ======

      → AiController.generateStream(keywords, templateType, additionalPrompt, model)
                                                            // AiController.java:143
        → new SseEmitter(300000L)                             // 创建 SSE 发射器（5分钟超时）
        → taskExecutor.execute(() -> {                        // 异步执行！不阻塞 Tomcat 线程
            aiService.generateStream(..., emitter)
          })
        → return emitter                                      // 立即返回，Tomcat 线程释放

       ====== 第二步：异步 AI 生成（线程池线程） ======

      → AiService.generateStream(keywords, templateType, additionalPrompt, model, emitter)
                                                            // AiService.java:114
        → buildMessages(keywords, templateType, additionalPrompt)
                                                            // AiService.java:207
          → aiTemplateMapper.selectOne(type='article')         // 查 ai_template 表拿 system_prompt
          → 组装消息列表:
              [
                {role: "system", content: "你是一位专业写手..."},
                {role: "user",   content: "请生成关于: Java Spring Boot..."}
              ]
        → deepSeekClient.streamChat(                         // 调用 DeepSeek API
            messages,
            onChunk,  // 收到文本块
            onComplete,// 生成完成
            onError   // 出错
          )

       ====== 第三步：DeepSeekClient 流式 HTTP 调用 ======

      → DeepSeekClient.streamChat(messages, onChunk, onComplete, onError)
                                                            // DeepSeekClient.java:105
        → restTemplate.execute(                               // 发 POST 请求到 DeepSeek
            uri, HttpMethod.POST,
            requestCallback → setHeaders(API Key, Bearer, JSON) + 写入请求体
            responseExtractor → {
              try (BufferedReader reader = new BufferedReader(    // 逐行读取流式响应
                       new InputStreamReader(response.getBody(), UTF-8))) {
                while ((line = reader.readLine()) != null) {
                  if (line.startsWith("data: ")) {
                    data = line.substring(6).trim()
                    if (data == "[DONE]") → onComplete.run()
                    else:
                      json = objectMapper.readTree(data)
                      content = json.at("/choices/0/delta/content").asText()
                      onChunk.accept(content)  ← 回调给 AiService
                  }
                }
              }
            })

       ====== 第四步：逐字推送到前端 ======

      DeepSeekClient 的 onChunk 回调触发
        → AiService 的 lambda: chunk → {
            emitter.send(SseEmitter.event()
              .name("message")
              .data(Map.of("content", chunk)))               // 推送给前端一个字
          }

      DeepSeekClient 的 onComplete 回调触发
        → emitter.send(SseEmitter.event().name("done").data(""))  // 通知前端完成
        → emitter.complete()                                      // 关闭 SSE 连接

      前端收到: event: message → data: {"content":"你"}
                event: message → data: {"content":"好"}
                ... 逐字追加 → 打字机效果
                event: done → 生成完毕

  流程 5：通知推送（RabbitMQ → WebSocket）

  任何业务代码调用发送通知（如积分变动、系统公告）
    → rabbitTemplate.convertAndSend(                        // 发到 RabbitMQ
        "smart.task.notification", notification)

      → NotificationRabbitListener.handleNotification(notification)
                                                            // NotificationRabbitListener.java:44
        → objectMapper.writeValueAsString(notification)      // 通知→JSON
        → webSocketHandler.sendToUser(userId, jsonMessage)   // 推送

          → NotificationWebSocketHandler.sendToUser(userId, message)
                                                            // NotificationWebSocketHandler.java:72
            → sessions.get(userId)                            // 从 ConcurrentHashMap 取连接
            → if (session != null && session.isOpen())
              → session.sendMessage(new TextMessage(message)) // WebSocket 推送
              → return true
            → return false                                    // 用户不在线

      WebSocket 连接建立时：
        前端 → new WebSocket("ws://localhost/ws/123?token=xxx")

        → WebSocketAuthInterceptor.beforeHandshake(request, response, handler, attributes)
                                                            // WebSocketAuthInterceptor.java:31
          → httpRequest.getParameter("token")                 // 取 URL 中的 token
          → jwtUtils.validateToken(token)                     // 验证 JWT
          → jwtUtils.getUserIdFromToken(token)                // 从 JWT 取用户 ID
          → attributes.put("userId", userId)                  // 存入会话属性
          → return true                                       // 放行

        → NotificationWebSocketHandler.afterConnectionEstablished(session)
                                                            // NotificationWebSocketHandler.java:38
          → session.getAttributes().get("userId")             // 取拦截器存的 userId
          → sessions.put(userId, session)                     // 存入在线 Map

  流程 7：接口限流

  任何请求到达时（在进入 Controller 之前）
    → RateLimitInterceptor.preHandle(request, response, handler)
                                                            // RateLimitInterceptor.java:42
      → 获取客户端 IP: request.getRemoteAddr()
      → redisTemplate.opsForValue().increment("rate:limit:" + ip)
                                                            // Redis INCR 原子+1
      → 如果 count == 1:
        → redisTemplate.expire(key, 10, SECONDS)             // 第一次请求设 10 秒过期
      → 如果 count > 10:
        → response.setStatus(429)                            // Too Many Requests
        → objectMapper.writeValueAsString(
            Result.failed("请求太频繁，请稍后再试"))          // 返回 JSON
        → return false                                       // 拦截请求
      → return true                                          // 放行

  流程 8：全局异常处理

  在任何 Controller 方法中抛出异常时
    → GlobalExceptionHandler 根据异常类型分发               // GlobalExceptionHandler.java

      ① BusinessException（自定义业务异常）
        → handleBusinessException(e)
          → log.warn("业务异常: code={}, message={}")
          → return Result.failed(code, message)

      ② 参数校验异常（@Valid 校验失败）
        → handleValidationException(e)
          → return Result.failed(ResultCode.VALIDATE_FAILED, e.getMessage())

      ③ Spring Security 权限不足
        → handleAccessDeniedException(e)
          → return Result.failed(ResultCode.FORBIDDEN)

      ④ 其他所有未捕获异常
        → handleException(e)
          → log.error("系统异常: ", e)
          → return Result.failed("服务器繁忙，请稍后重试")

  流程 10：幂等性检查（防止重复提交）

  请求到达被 @Idempotent 注解标记的 Controller 方法时
    → IdempotentAspect.around(joinPoint, idempotent)         // IdempotentAspect.java:49

      例如 ContentController.createContent() 上标注了 @Idempotent(message="内容正在保存中...")

      → RequestContextUtils.getRequest()                      // 取当前请求
      → request.getHeader("idempotent-token")                 // 从请求头取幂等令牌
      → 如果没传 → 直接放行 joinPoint.proceed()

      → redissonClient.getLock("idempotent:lock:" + token)   // 创建分布式锁
      → lock.tryLock(0, 5, SECONDS)                           // 立即尝试拿锁，最多等5秒
      → 没拿到锁 → return Result.failed("请勿重复提交")

      → redissonClient.getBucket("idempotent:key:" + token).isExists()
                                                            // 查 Redis 是否已处理
      → 已处理 → return Result.failed("请勿重复提交")
      → 未处理 → set(true, expireTime, SECONDS)               // 标记为已处理
      → joinPoint.proceed()                                   // 执行原方法

  一张图总结所有流程的走向

  请求进来
    │
    ├─① RateLimitInterceptor          ← IP限流（Redis INCR，10秒10次）
    │    └─ 超过 → HTTP 429
    │
    ├─② JwtAuthenticationFilter       ← JWT认证（取 Header 或 ?token）
    │    └─ 有效 → 设置 SecurityContextHolder
    │
    ├─③ OperationLogAspect            ← AOP记录操作日志
    │
    ├─④ IdempotentAspect              ← AOP幂等检查（仅 @Idempotent 方法）
    │
    ├─⑤ Controller → Service → Mapper ← 业务处理
    │    ├─ AuthController     → AuthService     → JWT + Redis
    │    ├─ AiController       → AiService       → DeepSeekClient → SseEmitter
    │    ├─ ContentController  → ContentService  → ContentMapper + ContentVersionMapper
    │    ├─ DashboardController→ DashboardService→ ContentMapper + UserMapper + Redis
    │    └─ NotificationController → NotificationService → DB
    │
    ├─⑥ 异常时 → GlobalExceptionHandler ← 统一封装 Result
    │
    └─⑦ 异步通知 → RabbitMQ → NotificationRabbitListener
                            → NotificationWebSocketHandler
                            → WebSocket 推送给前端
